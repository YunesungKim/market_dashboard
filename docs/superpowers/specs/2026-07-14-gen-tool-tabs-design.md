# 생성 도구 개선 (탭 + 모드 버튼 + 선택 배포 + 발행 관리) — 설계

작성일: 2026-07-14
대상: 증시 브리핑 생성 도구(로컬, `tools/`). 공개 아카이브는 변경 없음.

## 확정 사항 (브레인스토밍)

- 삭제(기존 발행 브리핑)는 **생성 도구(로컬 서버)** 에서 수행 — 서버가 `briefings.json`을 재작성 + git push. 공개 아카이브는 정적 유지.
- 카드별 체크박스 **선택 배포**. 미체크 카드는 **보드에 그대로 남김**. 체크박스 기본 **해제**.
- 모드 선택은 **세그먼트 버튼 2개**(체크박스 제거). 라벨: `Mode A: Serper API Only`, `Mode B: Summarized by LLM`. 기본 A.
- 생성 도구와 발행 관리는 **페이지 내 2개 탭**으로 전환.

## 변경 (모두 생성 도구 프론트 + 로컬 서버)

### 프론트엔드
1. **모드 세그먼트 버튼**: `useLlm` 체크박스·`modeBadge` 제거 → `Mode A: Serper API Only`(기본 활성) / `Mode B: Summarized by LLM` 상호배타 버튼. 활성 버튼이 곧 현재 모드 표시. 검색 시 활성 모드로 요청.
2. **카드별 체크박스 선택 배포**: 각 후보 카드에 체크박스(기본 해제). `배포 (N)`의 N = 체크된 카드 수(실시간). 배포 클릭 → 체크된 카드만 배포, 미체크는 보드 유지, 배포된 카드만 제거. 기존 카드 "삭제" 버튼 유지.
3. **진행 상태 박스**: `검색 중…`·`N건 배포 완료`·오류 등을 후보 카드 **위쪽** 별도 배너 박스(`.status-box`)로 이동.
4. **2개 탭**: `증시 브리핑 생성 도구`(후보 보드) / `발행된 브리핑 카드 관리`(발행 목록+삭제). 탭 클릭으로 패널 전환, 관리 탭 진입 시 목록 자동 로드.
5. 설정 모달(테마/모델)은 그대로 유지.

### 백엔드
- `POST /api/publish` → body `{cardIds:[...]}` 로 **선택 배포**. `cardIds` 키가 없으면 기존처럼 전체(하위호환), 있으면 해당 카드만 append/제거. 미선택은 보드 유지.
- `GET /api/briefings` → 현재 `briefings.json` 항목 반환.
- `POST /api/briefings/delete` → body `{ids:[...]}` → `briefings.json`에서 제거·재작성 + git push. 삭제 수 반환.
- `tools/store.py`에 `remove_briefings(path, ids) -> int`(삭제 개수) 추가.

## 수용 기준
- 모드 A/B 버튼 상호배타 토글, 기본 A, 검색이 활성 모드로 동작.
- 카드 체크박스 기본 해제, `배포 (N)`가 체크 수 반영, 체크된 것만 배포되고 미체크는 남음.
- 진행 문구가 카드 위 박스에 보임.
- 두 탭 전환 동작, 관리 탭에서 발행 목록 로드·선택 삭제 시 briefings.json 재작성+push.
- 백엔드 신규/변경 엔드포인트 테스트 통과, 전체 스위트 통과.
